Ubuntu 16.04 编译安装 LNMP 环境

发表于 2017-04-07 | 分类于 运维部署

个人习惯：源代码包放在 /usr/local/src 目录下，安装路径在 /usr/local下

Ubuntu是Vagrant+VirtualBox搭建

编译 Nginx

  • 下载 Nginx 源码包，我选择的是 1.10.3 稳定版
  • 解压源码包
  • 配置安装路径
  • 此时编译报错：没有 C 编译器

./configure: error: C compiler cc is not found

  • 再次配置安装路径，报错：缺少 PCRE 库

./configure: error: the HTTP rewrite module requires the PCRE library.

  • 下载 pcre 库源码并解压
  • 再次配置安装路径

报错：缺少 zlib 库
./configure: error: the HTTP gzip module requires the zlib library.

如上方法，下载 zlib 源码并解压

  • 再次配置安装路径

此时成功了 o(￣▽￣)d

  • 别忘了最后一步
  • 启动 nginx

我设置的该本地服务器的 IP 为 192.168.33.250

nginx 启动后，在chrome查看80端口，出现该页面，说明我已经成功安装nginx了o(￣▽￣)d

编译PHP

  • 下载源码包，我选择的是最新稳定版 7.1.4
  • 解压源码包
  • 配置安装路径

报错：缺少xml2-config，需要安装libxml2

configure: error: xml2-config not found. Please check your libxml2 installation.

继续配置安装路径，报错：缺少openssl

configure: error: Cannot find OpenSSL’s <evp.h>

继续配置安装路径，报错：

configure: error: Cannot find OpenSSL’s libraries

解决方案：http://stackoverflow.com/questions/39919283/configure-error-cannot-find-openssls-libraries-ubuntu-16-04

继续配置安装路径，缺少gd库，但是我找了下，并没有php7.1-gd这个库，退而且其次，安装了jpeg和png的库

configure: error: png.h not found.

继续配置安装路径，这一次终于成功了😄

然后优雅的重启下Nginx 和 php-fpm

报错：WARNING: Nothing matches the include pattern ‘/usr/local/php/etc/php-fpm.d/*.conf’ from /usr/local/php/etc/php-fpm.conf at line 125.

很简单，在/usr/local/php/etc/php-fpm.d/ 目录下，把 www.conf.default 重命名或复制份，以 conf 为后缀即可

然后再启动php-fpm。
